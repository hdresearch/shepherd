Guides are **task recipes**: each does one named job and keeps one shape —
job, prerequisites, steps, expected result, and what to do when it fails.
They assume you have already met Shepherd (start with the
[tutorial](../tutorials/first-shepherd-app.md) if not). The guide set is
owner-led — curated by hand, not auto-generated — and no guide is promoted
until its commands run against shipped surfaces. The planned launch set:

- Configure a provider
- Run a workflow
- Test Shepherd code
- Route tasks to models
- Debug your first run

Early drafts of two of these exist as scaffolds in the reviewer build; the
launch list itself stays owner-curated and may change before promotion.
